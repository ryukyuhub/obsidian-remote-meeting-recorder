
# Remote Meeting Recorder 機能拡張要請書

> [!info] 概要
> Obsidianプラグイン **Remote Meeting Recorder（RMR）** に対し、**保存済み音声ファイルを右クリックしてその場で文字起こしできる機能**の追加を要請します。

| 項目 | 内容 |
| --- | --- |
| 宛先 | Ryukyu HUB Inc.（RMRプラグイン開発担当） |
| 起票日 | 2026-07-06 |
| 対象プラグイン | Remote Meeting Recorder `v0.3.1` |
| 対象環境 | Windows 11 / macOS（Desktop only） |
| 優先度 | 中〜高 |

---

## 1. 背景・目的

現在RMRは、**録音停止時（`transcribeOnStop`）に限って**自動で文字起こしを行い、ノートへ埋め込みとトランスクリプトを挿入する。しかし以下のニーズに応えられていない。

- 過去に録音した `.m4a` を**後から文字起こしし直したい**（当時 `small` モデルで精度が不十分だった等）。
- RMR以外の手段で入手した音声ファイル（`.mp3` 等）を**文字起こししたい**。
- 文字起こしに失敗／未実行だった録音を**個別にやり直したい**。

そこで、**ファイルエクスプローラや埋め込み音声を右クリック → 「文字起こし」**という導線を追加し、録音フローと切り離して任意の音声を文字起こし可能にしたい。

---

## 2. 現状仕様（確認済みの事実）

要請の前提として、現行 `v0.3.1` を調査した結果を記す。

- whisper.cpp 一式を**プラグイン内に同梱済み**
	- 実行体: `native/whisper/Release/whisper-cli.exe`（＋ `whisper.dll` / `ggml*.dll` 等）
	- モデル: `ggml-base.bin` / `ggml-small.bin` / **`ggml-large-v3-turbo-q5_0.bin`**
	- ソース: `main.js` 内に `transcribeFile` 関数、whisper バイナリ／モデルの自動DL処理（`ggml-org/whisper.cpp` releases・HuggingFace）を確認
- 設定（`data.json`）
	- `transcribeOnStop: true` / `whisperCppModel: "small"` / `transcribeLanguage: "ja"`
	- `whisperCppBinPath` / `stateDir` は空（同梱パスへフォールバック）
- **制約**: `whisper-cli.exe` の入力は **16kHz WAV 前提**。録音停止時は内部WAVを直接渡せるため問題ないが、**保存済み `.m4a` を後から渡すにはデコード（WAV変換）が必要**。現状プラグインには **ffmpeg 等のデコーダが同梱されていない**。

> [!warning] 技術的な肝
> 「右クリック文字起こし」の実現には、**任意の音声 → 16kHz mono WAV へのデコード手段**をプラグインが内部で持つことが必須。ここが本要請の中心的な追加要素。

## 4. 詳細要件

### 4.1 対応音声形式
- [ ] 最低限: `.m4a`, `.mp3`, `.wav`
- [ ] 可能なら: `.mp4` / `.mov` の音声トラック, `.aac`, `.flac`, `.ogg`

### 4.2 音声デコード（最重要）
- [ ] 任意形式 → **16kHz / mono / 16bit WAV** への変換を内部実装
- [ ] 実装案（いずれか）
	- **ffmpeg を同梱**し `native/` から呼び出す（録音バイナリと同様の配布方式）
	- miniaudio 等の**デコーダをwhisperラッパへ内蔵**
	- Windows: Media Foundation / macOS: AVFoundation を利用
- [ ] デコーダ不在時は**明確なエラー**と導入案内を表示（無言失敗にしない）

### 4.3 モデル選択
- [ ] 実行時に**モデルを選択**できる（`base` / `small` / `large-v3-turbo`）
	- 既定は設定 `whisperCppModel`、右クリック実行時に上書き選択可
- [ ] 未DLモデルを選んだ場合は自動DL or 明確な警告

### 4.4 言語
- [ ] 既定は `transcribeLanguage`（現状 `ja`）。実行時に言語指定 or `auto` を選べると尚可

### 4.5 実行体験
- [ ] **進捗表示**（Notice / ステータスバー等）と**キャンセル**
- [ ] 長時間音声でもUIを**ブロックしない**（バックグラウンド実行）
- [ ] 既存 `transcribeFile` ロジックを**再利用**し、入り口（デコード＋メニュー）のみ追加する設計が望ましい

### 4.6 冪等性・安全性
- [ ] 同一音声に対する**再実行時の重複挿入を防止**（既存トランスクリプトを検出し「置換／追記／中止」を選択）
- [ ] **原本音声は変更しない**（読み取りのみ。WAVは一時ファイルとして生成・後始末）

---

## 5. 非機能要件

- [ ] **完全ローカル処理**（会議音声を外部送信しない。機密保持のため必須）
- [ ] オフラインで動作（モデルDL済みなら）
- [ ] Windows / macOS 両対応
- [ ] 既存の録音・停止時文字起こしの挙動を**壊さない**（後方互換）
