# DRM対策 Spike（Core Audio タップ移行の検証）

`macOS DRM対策 CoreAudioタップ移行 実装レポート.md` §6 の検証を、**本番コードに触れず**実機で潰すための最小試作。
本番の `sysrec.swift` とは完全に独立（別バイナリ・別署名）。

## ビルド

```sh
cd native/sysrec/spike
sh build.sh        # → ./tap と ./micsck を生成（ad-hoc 署名込み）
```

## 2つの試作

| バイナリ | 何を試すか | 検証項目 |
|---|---|---|
| `./tap`    | Core Audio プロセスタップでシステム音声を取得（画面キャプチャなし） | ①②③⑤⑥⑦ |
| `./micsck` | マイク単独の SCStream（本番 mic 経路の最小再現）                    | ④（スコープ確定・最重要） |

---

## 手順 A：`tap`（本命 ①）

```sh
./tap --seconds 20 --out /tmp/tap-spike.wav
```

1. 実行すると**初回はオーディオ録音の許可プロンプト**が出るはず（→ ③ の記録対象）。許可する。
2. 「録音中…」表示の間に、ブラウザで **Netflix 等の DRM 動画を再生**する。
3. 20 秒後に自動停止。`/tmp/tap-spike.wav` を再生して音を確認。

**観察して記録すること**
- **①（本命）**: 再生中に映像が**黒くならなかったか**。← これが移行の成否。
- **②**: `tap-spike.wav` に**システム音声が入っているか**（ログの「ピーク」も参照。無音なら ~0）。
- **③**: どの権限が要求されたか（「画面収録」ではなく「オーディオ録音」等）。プロンプト文言・出るタイミング。
- **⑤（任意）**: 録音中に出力先を 内蔵↔BT↔AirPlay 切替 → 無音化/クラッシュしないか。
- **⑥**: ログ冒頭の ASBD と「除外プロセス」。自アプリ音除外は本番統合で本確認（ここでは配線のみ）。
- **⑦**: ログの `⑦ ASBD:` 行（Hz / ch / bits / float / interleaved）。既存 WriterBox と整合するか。

`✗` 行（OSStatus 付き）が出たら、その行をそのまま共有してください（API 誤用を直します）。

---

## 手順 B：`micsck`（スコープ確定 ④・最重要）

```sh
./micsck --seconds 20
```

1. 「mic 単独 SCStream 稼働中…」の間に、**同じく Netflix を再生**する。
2. 20 秒後に自動停止。

**観察して記録すること**
- **④**: この「マイクだけ」の SCStream 稼働中に、映像が**黒くなったか / ならなかったか**。

| ④ の結果 | 意味 | スコープ |
|---|---|---|
| **黒くなった** | マイク単独の SCK でも「画面録画中」になる | マイクも SCK 撤去（AVAudioEngine へ）＝**大** |
| **黒くならない** | DRM トリガはシステム音声側の SCK だけ | マイクは SCK 継続・システム音のみタップ化＝**小** |

この 1 点で本実装の工数が倍近く変わります（レポート §5）。

---

## 片付け

試作が作る集約デバイスは private かつ停止時に破棄しています（`AudioDeviceStop → DestroyIOProcID → DestroyAggregateDevice → DestroyProcessTap` の順）。
もし異常終了でデバイスが残った場合は、`AudioMIDI 設定.app` に private 集約デバイスが残っていないか確認してください（通常は残りません）。
