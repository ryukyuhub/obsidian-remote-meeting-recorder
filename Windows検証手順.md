# Windows 実機検証手順（リファクタ調査 R4）

Windows 側は実装・修正（0.6.0 の無音対策 / 0.7.0 の取り込み時正規化）を**実機で一度も通せていない**。
この手順は Windows 機を持つ人（＝ユーザー）に 1 往復だけ協力してもらい、未検証項目を一括で潰すためのもの。

## やること（約 2 分＋録音 1 本）

### 1. 診断スクリプト（約 75 秒）

1. Windows の Obsidian でプラグインを最新（0.8.0 以降）に更新
2. `Ctrl+Shift+I` で開発者ツール → Console タブ
3. リポジトリの [dev/win-audio-diag.js](dev/win-audio-diag.js) の中身を全部コピーして Console に貼り付け → Enter
4. 画面の指示に従う:
   - 「何か喋ってください」→ マイクに向かって話す
   - 「音楽/動画を鳴らし続けてください」→ YouTube 等を**通常音量**で再生
   - 「音量を半分に下げて」→ Windows のタスクバーの音量を下げる（鳴らしたまま）
   - 「出力デバイスを切り替えて」→ スピーカー⇔イヤホンを切替（無ければスキップ可）
5. 最後に出る **1 行の JSON をコピーして報告**

### 2. 実録音 1 本（1 分）

1. 録音ビューで Source=**両方**・AutoGain=**オン**のまま録音開始
2. 音楽/動画を 30 秒鳴らしながら、ときどき喋る
3. 停止 → 保存されたファイルを聞いて、**システム音と自分の声の両方が聞こえるか**確認
4. 気になる点（小さい・無音・途切れ）があればファイルごと報告

## この結果で確定すること（→ リファクタリング調査レポート §4.3）

| 診断の項目 | 確定する未検証項目 |
|---|---|
| `[A] mediaRecorder.mp4_aac` | Windows で .m4a 保存できるか（false でも .webm フォールバックで動作はする） |
| `[B] micPeakDb` | マイク取得と AGC 入力の健全性 |
| `[C] loopback` | WASAPI ループバック取得（0.6.0 の無音対策の効果） |
| `[D] normalize20s` | **0.7.0 の取り込み時正規化が実信号で -16 dBFS 近傍へ収束するか**（out が -18〜-14 なら合格） |
| `[E] volumeChange15s` | 出力音量を下げても out が安定しているか（in は下がって良い・**out が追従して下がるなら問題**） |
| `[F] deviceSwitch15s` / `loopbackEnded` | 出力デバイス切替でループバックが生きるか（ended=fail-safe 発動 / in が -inf のままなら「黙って無音」＝macOS R0 と同種の対処が必要） |

## 判定の目安

- **[D] の out が -18〜-14 dBFS** → 取り込み時正規化は機能。合格。
- **[E] で in が下がり gain が上がって out がほぼ一定** → 「録音レベルを出力音量から独立させる」目的を達成。
- **[F] で trackState が "ended"** → 現行の fail-safe（自動 finalize）が働く。録音は途切れるが無音記録は残らない＝許容。
  **in が -inf のまま ended にもならない** → Windows にも追従処理が必要（新 issue として起票）。
